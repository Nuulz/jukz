# jukz Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline,
> chosen for solo same-session build). Steps use checkbox (`- [ ]`) syntax.

**Goal:** Build the jukz Fabric 1.21.1 mod scaffold plus the real, unit-tested
deterministic core (WorldID, host election with fencing, handshake codec, in-memory
registry, local TCP relay), with network-dependent adapters (DHT/NAT/sync) coded
against real libraries but flagged for live-network testing.

**Architecture:** A Gradle build with two source sets/modules so the deterministic
core compiles and tests **without** Minecraft/Loom on the classpath: a pure-Kotlin
`core` module (election, handshake, registry interface + in-memory impl, relay, world
id model, clock) tested with JUnit5, and a `fabric` module that depends on `core` and
wires it into Minecraft via Fabric API + a client mixin. Network adapters
(`MldhtWorldRegistry`, `IceTransport`, `JGitWorldSync`) live in `fabric` (or a
`net` source set) and are flagged TODO for live testing.

**Tech Stack:** Kotlin, Gradle (Kotlin/Groovy DSL), Fabric Loom 1.7, Fabric API
0.116.12+1.21.1, yarn 1.21.1+build.3, fabric-language-kotlin, JUnit5, kotlinx-coroutines,
the8472/mldht (JitPack), ice4j, netty-codec-native-quic, JGit 7.x.

## Task 3: ClaimToken ordering (TDD)

**Files:** `election/ClaimToken.kt`; test `ClaimTokenTest.kt`.

Contract:
```kotlin
data class ClaimToken(
    val hostGeneration: Long,   // u64 fence; compared unsigned-safe (non-negative by construction)
    val claimEpochMillis: Long, // tiebreak only
    val nodeId: NodeId          // 16 bytes; final tiebreak, unsigned lexicographic
) : Comparable<ClaimToken> {
    override fun compareTo(other: ClaimToken): Int { /* generation, then millis, then nodeId */ }
}
```

- [ ] Tests: higher generation wins regardless of millis/nodeId; equal generation → higher
      millis wins; equal gen+millis → lexicographically-greater nodeId wins; equal all → 0;
      a true tie requires identical nodeId (different installs never tie).
- [ ] Implement; verify; commit.

## Task 4: InMemoryWorldRegistry CAS+TTL (TDD)

**Files:** `discovery/WorldRecord.kt`, `WorldRegistry.kt`, `InMemoryWorldRegistry.kt`;
test `InMemoryWorldRegistryTest.kt`.

Contract:
```kotlin
sealed interface PublishResult {
    data class Published(val record: WorldRecord) : PublishResult
    data class Rejected(val current: WorldRecord) : PublishResult   // existing token >= candidate
}
interface WorldRegistry {
    suspend fun publishIfNewer(record: WorldRecord): PublishResult
    suspend fun lookup(worldId: UUID): WorldRecord?
    suspend fun withdraw(worldId: UUID, token: ClaimToken)
}
```

- [ ] Tests: first publish → Published; lower/equal token → Rejected(current); strictly higher
      token → Published (overwrite); lookup returns latest; withdraw only if token matches
      current; TTL expiry (drive via FakeClock) → lookup returns null after `ttlMs`.
- [ ] Implement (backed by FakeClock-driven map); verify; commit.

## Task 5: Handshake messages + codec (TDD)

## Task 6: HostElection (R1-R14) (TDD)

**Files:** `election/ElectionConfig.kt`, `ElectionOutcome.kt`, `HostElection.kt`;
test `HostElectionTest.kt`. Uses `WorldRegistry` + `JukzClock`.

- [ ] Tests (each maps to numbered rules):
      - R1-R3: uncontested claim → BecameHost; record holds candidate token.
      - R4: existing higher token present → BecameGuest(redirect=current.endpoint).
      - R5/R6: stale lower record present → CAS overwrite, BecameHost.
      - R7: determinism — two crafted competing claims, higher token wins, loser BecameGuest.
      - R8-R11: ghost (no heartbeatSeq advance over LIVENESS_TIMEOUT via FakeClock) → Ghost.
      - R12-R13: takeover publishes generation+1; revived ghost with lower gen → demoted.
      - R14: `validateIncoming(token)` rejects strictly-lower generation (STALE_TOKEN).
- [ ] Implement; verify; commit.
